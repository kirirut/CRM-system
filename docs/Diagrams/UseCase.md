# 2 Варианты использования

<a name="sign_in"/>

## 2.1 Войти в систему

**Описание.** Позволяет пользователю войти в учетную запись.  
**Предусловия.** Пользователь выбрал вход в систему.  
**Основной поток.**
1. Приложение отображает окно входа;
2. Пользователь вводит учетные данные;
3. Приложение проверяет данные и присваивает статус "зарегистрирован";
4. Приложение отображает интерфейс пользователя;
5. Вариант использования завершается.

**Альтернативный поток.**
1. Если введены неверные данные, приложение выводит сообщение об ошибке;
2. Пользователь может повторить ввод.

<a name="sign_up"/>

## 2.2 Регистрация пользователя

**Описание.** Позволяет пользователю создать учетную запись.  
**Предусловия.** Пользователь выбрал регистрацию.  
**Основной поток.**
1. Приложение отображает форму регистрации;
2. Пользователь вводит имя и другие данные;
3. Приложение проверяет уникальность имени;
4. Приложение создаёт учетную запись;
5. Приложение присваивает статус "зарегистрирован";
6. Вариант использования завершается.

**Альтернативный поток.**
1. Если имя уже существует, приложение запрашивает другое имя.

<a name="add_order"/>

## 2.3 Добавить заказ

**Описание.** Позволяет зарегистрированному пользователю добавить новый заказ.  
**Предусловия.** Пользователь вошел в систему.  
**Основной поток.**
1. Пользователь выбирает "Добавить заказ";
2. Приложение отображает форму заказа;
3. Пользователь вводит данные заказа;
4. Приложение сохраняет заказ в базе;
5. Вариант использования завершается.

<a name="view_orders"/>

## 2.4 Просмотреть заказы

**Описание.** Позволяет пользователю просмотреть список заказов.  
**Предусловия.** Пользователь вошел в систему.  
**Основной поток.**
1. Приложение получает список заказов;
2. Приложение отображает заказы в интерфейсе;
3. Вариант использования завершается.

<a name="remove_order"/>

## 2.5 Удалить заказ

**Описание.** Позволяет пользователю удалить заказ.  
**Предусловия.** Пользователь вошел в систему.  
**Основной поток.**
1. Пользователь выбирает заказ для удаления;
2. Приложение удаляет заказ из базы;
3. Приложение обновляет интерфейс;
4. Вариант использования завершается.

<a name="sign_out"/>

## 2.6 Выйти из системы

**Описание.** Позволяет пользователю выйти из учетной записи.  
**Предусловия.** Пользователь вошел в систему.  
**Основной поток.**
1. Пользователь выбирает "Выйти";
2. Приложение скрывает интерфейс пользователя;
3. Приложение отображает окно входа;
4. Вариант использования завершается.
