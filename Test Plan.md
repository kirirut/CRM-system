## Введение

Данный документ описывает план тестирования CRM-системы ["CRM-system"](https://github.com/kirirut/CRM-system). Документ предназначен для людей, выполняющих тестирование данного проекта. Цель тестирования — проверка соответствия реального поведения программы ожидаемому поведению, которое описано в [требованиях](https://github.com/kirirut/CRM-system/blob/main/Documents/Requirements.md).

<a name="items"/>

## Объект тестирования

В качестве объектов тестирования выделяются следующие функциональные возможности CRM:

* Вход в систему;
* Регистрация пользователя;
* Добавление заказа;
* Просмотр заказов;
* Удаление заказа;
* Выход из системы.

<a name="quality"/>

## Атрибуты качества

1. Функциональность:
    - функциональная полнота: система должна поддерживать все заявленные функции;
    - функциональная корректность: каждая функция должна выполняться корректно;
2. Удобство использования:
    - интерфейс должен быть понятен пользователю, все кнопки и поля иметь понятные подписи;
3. Надежность:
    - система должна корректно обрабатывать ошибки и некорректные данные.

<a name="risk"/>

## Риски

* Ошибки при сохранении или удалении заказов;
* Некорректная обработка входа или регистрации пользователя;
* Потеря данных при сбоях системы.

<a name="features"/>

## Аспекты тестирования

В ходе тестирования проверяется реализация основных функций CRM:

### Вход в систему
* Отображение окна входа;
* Проверка корректности учетных данных;
* Обработка неверных данных.

### Регистрация пользователя
* Отображение формы регистрации;
* Ввод данных нового пользователя;
* Проверка уникальности имени;
* Создание учетной записи.

### Добавление заказа
* Отображение формы добавления заказа;
* Ввод данных заказа;
* Сохранение заказа в системе;
* Проверка корректности добавленного заказа.

### Просмотр заказов
* Отображение списка всех заказов;
* Корректное отображение данных заказа.

### Удаление заказа
* Выбор заказа для удаления;
* Корректное удаление заказа из системы;
* Обновление интерфейса.

### Выход из системы
* Корректное завершение сеанса пользователя;
* Отображение окна входа после выхода.

<a name="conclusion"/>

## Выводы

Данный план тестирования позволяет проверить основной функционал CRM-системы. Успешное прохождение всех тестов не гарантирует полной работоспособности на всех платформах и устройствах, однако позволяет убедиться, что ключевые функции работают корректно. После завершения всех этапов разработки тестирование должно быть повторено согласно данному плану.
